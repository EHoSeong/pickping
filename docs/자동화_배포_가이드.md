# PickPing — 자동화 배포 가이드 (GitHub Pages + Actions)

이 가이드대로 한 번만 세팅하면, **매일 자동으로 사이트가 재빌드·배포**됩니다.
이후엔 손 안 대도 가격·순위가 갱신돼요. (님이 원한 "구축 후 방치")

## 전체 그림

```
seeds.json ──(로컬)──▶ keywords.json ──커밋──▶ GitHub
                                                  │
                              매일 06:00 KST, GitHub Actions 자동 실행
                                                  │
              Secrets(네이버 키) → config.json 생성 → build.py → GitHub Pages 배포
```

- **키워드 확장**(자동완성 크롤링)은 로컬에서: 클라우드 IP 차단 방지를 위해 로컬에서 돌리고 커밋
- **빌드·배포**(상품 데이터 갱신)는 GitHub Actions가 매일 자동

## STEP 1. 로컬에서 키워드 만들고 커밋

```bash
copy seeds.example.json seeds.json      # 관심 카테고리 시드 20~50개로 편집
python find_keywords.py --depth 2       # keywords.json 대량 생성
```
→ `seeds.json`, `keywords.json`을 커밋합니다. (`config.json`은 커밋 금지 — 키는 Secrets로)

## STEP 2. GitHub 저장소 생성 & 푸시
- 새 저장소를 만들고 이 `pickping` 폴더를 푸시합니다.
- `.github/workflows/deploy.yml` 이 함께 올라갑니다.

## STEP 3. GitHub Pages 활성화
저장소 → **Settings → Pages → Build and deployment → Source: `GitHub Actions`** 선택.

## STEP 4. Secrets 등록 (여기가 핵심)
저장소 → **Settings → Secrets and variables → Actions → New repository secret** 로 아래 등록:

| Secret 이름 | 값 | 필수 |
|---|---|---|
| `NAVER_CLIENT_ID` | 네이버 검색 API Client ID | ✅ |
| `NAVER_CLIENT_SECRET` | 네이버 검색 API Client Secret | ✅ |
| `SITE_BASE_URL` | Pages 주소 (예: `https://내계정.github.io/pickping`) | ✅ |
| `SITE_NAME` | 사이트 이름 (예: `오늘의 추천픽`) | 선택 |
| `COUPANG_PARTNERS_ID` | 쿠팡파트너스 ID | 선택 |

> `SITE_BASE_URL`은 sitemap·canonical에 쓰이므로 **정확한 Pages 주소**를 넣어야 검색 색인이 제대로 됩니다.

## STEP 5. 실행
- **Actions** 탭 → `Build and Deploy` → **Run workflow**(수동 1회 실행)
- 이후엔 매일 06:00(KST) 자동 실행 + main 브랜치 푸시 때도 실행

## STEP 6. 검색엔진 등록 (색인 유도)
1. **구글 서치콘솔**에 사이트 등록 → `sitemap.xml` 제출
   (주소: `SITE_BASE_URL/sitemap.xml`)
2. **네이버 서치어드바이저**에도 동일하게 등록 (네이버 유입도 노림)

## 운영 팁
- **키워드 늘리기**: 주기적으로 로컬에서 `find_keywords.py --merge` 돌려 keywords.json 확장 후 커밋
- **cron 주기 변경**: `deploy.yml`의 `cron: "0 21 * * *"` 수정 (UTC 기준)
- **수익화**: 색인·트래픽이 붙기 시작하면 애드센스 승인 신청 → 광고 코드를 `templates/base.html`에 삽입

## 자주 막히는 곳
- **배포는 됐는데 페이지가 깨짐** → `SITE_BASE_URL`이 실제 Pages 주소와 다른 경우. Secret 재확인.
- **build 실패(KeyError: keywords)** → `keywords.json`을 커밋 안 함. STEP 1 재확인.
- **자동완성 크롤링이 Actions에서 실패** → 의도된 설계. 키워드 생성은 로컬에서만.
